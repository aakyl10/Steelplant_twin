# Diploma Demo Evidence Checklist

This checklist defines the practical evidence to collect for the Astana IT
University diploma defense. The project is a simulation-based proof of concept
for a virtual steel plant compressed-air system:

```text
virtual sensors -> MQTT -> InfluxDB -> Grafana -> ML
```

Do not fabricate screenshots. Capture each image from a real local run of the
repository. This evidence supports a practical software demonstration only; it
does not prove real industrial deployment or measured industrial accuracy.

Recommended location for newly captured demo evidence:

```text
docs/evidence/
```

Keep existing evidence and ML outputs in place. In particular, do not remove
files from `docs/screenshots/`, `docs/ml/`, or `data/processed/`.

## Defense Notes

- Present the evidence in pipeline order: Docker, MQTT subscriber, publisher,
  InfluxDB, Grafana, ML outputs, tests, lint.
- State clearly that telemetry is generated by `simulator.generator` and is not
  collected from a real steel plant.
- Explain that SEC is a simulation proxy used to demonstrate data flow,
  dashboarding, and ML regression workflow.
- For ML results, describe Linear Regression as the baseline and Random Forest
  as the main model.
- Do not describe the metrics as real industrial accuracy. Describe them as
  evidence that the synthetic pipeline and deterministic ML experiment run
  correctly.
